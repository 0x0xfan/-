# 信息流视频生成规则

这个文件是项目规则库。以后新增、修改视频生成规则，优先写在这里，再同步到代码和测试。规则不要依赖某一次聊天上下文，也不要只针对某一批文案写死。

## 总原则

- 工具要适配各种类型文案，不能写死某个行业、某几个词、某一条文案。
- 先读文案语义，再决定字幕、大字、配图、素材、音效的时间线。
- 时间线顺序必须是：文案 -> TTS 配音 -> 真实音频时间线 -> 字幕/大字/配图/素材。
- 执行批量生成或导出前必须先做 TTS 预检：`tts.config.yaml` 的 `active` 不能为空，所选 provider 必须是当前代码已接入的 TTS，且必填的 key/appid/voice 等配置齐全。
- TTS 预检必须在正式生成前发起一次极短试音，确认能真实生成音频；预检音频只用于检查，不进入导出目录。
- TTS 未接入、配置不完整、服务端报错或配额耗尽时，必须立即停止生成；不能自动降级为无口播、估算时长或只带 BGM 的版本。
- 禁止用 `--provider=`、空 `active` 或其他方式绕过 TTS 执行生成；`--no-render` 也仍然必须先生成真实配音，因为字幕和时间线要以真实音频为准。
- 只有素材结构检查类命令（例如 `--inspect`）可以在没有 TTS 的情况下运行，因为它不生成 timeline、字幕、配音或 mp4。
- 单个视频内字幕样式固定使用一个样式。
- 输出成品目录只保留 mp4；中间文件留在运行目录，不放进最终导出目录。

## 字幕规则

- 字幕放在视频底部安全区域。
- 字幕以真实配音时间线为准，不能比语音提前。
- 优先使用 TTS 返回的词/句时间戳；没有时间戳时，按句生成配音并用每句真实音频时长分配字幕。
- 字幕尽量单行，必要时最多两行；必须自动缩小字号避免超出画幅。
- 同一个视频只使用一个字幕样式，不在同一条视频里随机切换。

## 大字规则

- 大字不能做成难看的白底卡片；默认应是透明大字、花字、贴纸式文字。
- 正式生成运行时不能读取剪映预设目录；剪映大字只能通过项目内置 `assets/styles/jianying_big_text_presets.json` 和已复制到项目内的字体/音效资产使用。
- 缺少本地素材时，不能再生成收益看板、低成本、复制粘贴等 PNG 卡片兜底；只能生成透明大字/文本序列，由项目内置大字样式渲染。
- 大字必须在安全区域内，不能出画幅，不能遮挡底部字幕。
- 连读顿号、斜杠、竖线、以及/还有/和/与/及 等形成的枚举句，应识别为“列表序列”。
- 列表序列不要只当普通字幕处理，应拆成多个短词逐个出现，形成节奏感。
- 列表序列是通用结构规则，不写死具体词。例如任何 `A、B、C、D` 都应尝试转成逐词大字序列。
- 抽出的序列词要短，去掉“比如、例如、包括、搜索、这些词、等等”等连接词和尾词。
- 负向卖点并列句必须优先生成大字序列，例如“不用囤货、不用发快递、不用露脸拍视频”；如果同句后面出现“低成本副业/低成本项目”等总结词，要追加为最后一个大字节奏点。
- 大字只显示关键词或短词组，不能拿大半句当大字，更不能显示被截断的不完整句子；抽不到关键词时宁可不补大字。
- 在结尾钩子引导之前，搜索词、资料类型、卖点等长顿号列表写死用大字逐个输出，不要用同行案例或其他图片抢画面。
- 抽不出合适短词时，不要硬生成大字。

## 画面填充规则

- 轨道1是背景基础画面，不能被黑底、白卡、大面积不透明图层盖住。
- 轨道2素材优先级：本地明确匹配素材 > 真实物体配图 > 文本序列大字 > 长空窗补充大字。
- 连续超过 3 秒没有轨道2素材时，必须补充画面元素。
- 3-8 秒空窗：补 1 个短语义大字/贴纸。
- 8 秒以上空窗：补多个语义大字，按时间逐个出现。
- 空窗补充应从当前无素材片段文本里抽取短标签，不能写通用废话，如“重点来了”“记住这句”。
- 空窗补充规则要通用，不依赖某个行业固定词。

## 本地素材匹配规则

- `其他素材` 按文件夹名做语义匹配，不按单个媒体文件名乱匹配。
- 当一句话命中某个本地素材目录，而且这句话里还能对应到该目录下多个具体素材时，不要只放一个；应优先挑 2-3 个代表素材，在这句话时长内逐个出现，可累计保留在屏幕上。
- 操作录屏类素材只用于明确操作步骤，例如上传图片、上文案、改价格、发货方式。
- 操作录屏不能用于抽象概念句，例如价格战、复制粘贴、自动发货策略、泛泛的方法论。
- 赠送资料图只能在视频结尾 CTA 附近出现，不能出现在前面或中间；必须有真实时间线，且片段开始时间已经进入结尾窗口才允许匹配，长片段从中段开始但结束拖到结尾也不能使用赠送资料图。
- 视频开头、实操步骤出现之前，如果文案提到“拿到好结果、跑通流程、出单、成交、到账、收益就有这么多”等结果证明，必须优先使用收益图。
- 搜索/找品/看同行阶段，明确提到“同行、案例、卖得好、想要数、浏览量”等判断同行的内容，才使用“闲鱼同行案例”素材。
- 结尾引导前的资料清单如果是长顿号列表，例如“选品库、爆品库、自动发货软件、运营教程、知识库、违规词库”，应从赠送资料图里选 2-3 个最相关的图显示，不能空画面；如果没有足够匹配图片，再退回大字序列。
- 收益截图要保持可读；多张收益截图连续出现时，用大图层叠/累计展示，不要缩成难读的小宫格。
- 视频类轨道2素材在视频中出现和消失都用硬切，不做淡入、滑入、缩放等转场。

## 自动配图规则

- 自动生成图片只用于物理世界真实存在的物体，例如矿泉水、手机、电脑、快递盒、汽车、具体产品等。
- 抽象名词不生成图片，例如虚拟资料、网盘链接、资料包、订单、PPT 模板、教程、流程、方法、收益等。
- 抽象名词优先用大字、字幕、贴纸或本地素材表达。
- 自动配图必须是透明前景或干净贴纸，不允许带大面积黑框/白框盖住轨道1。
- 如果没有真实图片生成 API key，兜底图也不能是白底卡片，应尽量是透明对象贴纸。
- 批量生成不能依赖聊天窗口的临时生图能力；项目需要可配置的图片生成入口，保证以后批量文案也能自动跑。

## 音效规则

- 音效不要乱放。只有大字特效、重点素材出现、明确转场时才考虑音效。
- 没有大字或图片素材出现时，不应该凭空出现音效。
- 本地项目内置音效可直接调用，不要求素材文件夹每次单独放音效。
- 音效音量要低于人声，不能影响听清配音。

## BGM 规则

- 背景音乐每条视频随机/轮换使用，不能批量生成时全都同一首。
- BGM 音量保持低，不压过配音。

## 渲染与性能规则

- 有 NVIDIA NVENC 时优先用 GPU 编码；不可用时回退 CPU。
- GPU 主要加速编码，不负责 TTS 网络请求和 Python 逐帧合成。
- 后续优化方向：TTS 并发请求、减少逐帧 Python 合成、复用生成素材。

## 需要继续进化的规则

- 通用“真实物体名词”识别目前还需要增强，不能长期只靠少量内置候选。
- 通用语义短标签抽取需要持续从实际视频反馈中改进。
- 大字样式需要继续优化，目标是更接近剪映花字/短视频常见样式，而不是普通描边字。

## 2026-05-07 反馈迭代：大字与音效

- 大字语义必须完整。不能为了适配宽度把“晚上只有一两个小时”截成“晚上只有一两”，也不能把“拿到资料后交付”截掉开头或结尾；应该优先缩小字号、拆成最多两行，再保证安全区内完整显示。
- 没有其他图片/视频素材时，大字默认放在画面中央或略微偏上，避开底部字幕；不要直接沿用剪映预设里的边缘位置。
- 大字必须在安全区内完整显示，不能贴边、出画幅或被旋转/阴影挤出画面。
- 音效必须跟可见画面元素绑定。没有大字或素材出现时不能凭空响音效。
- 一个大字/素材出现动作最多触发一个短音效；剪映预设自带音效和项目独立音效不能叠加成连续多响。
- 音效需要全局去重和降音量，避免短时间内多次响、压过人声。
